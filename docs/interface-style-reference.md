# Interface Style Reference

Use this as the visual direction for future interface refinement. The goal is a clean, structured UI with a polished, whimsical vector-scribble treatment: real app layouts first, intentional wobble second.

## Style Principles

- Build from clean, grid-aligned interface shapes: buttons, search bars, panels, cards, inputs, toggles, and navigation controls should still read like precise software UI.
- Apply a uniform scribble or wobbly vector treatment over the clean geometry instead of drawing messy freehand whiteboard doodles.
- Use crisp black fineliner-style strokes with a consistent stroke weight.
- Keep paths closed, continuous, and deliberate; the wobble should feel generated by a vector brush, not like loose marker sketching.
- Favor high contrast on stark white or soft off-white backgrounds.
- Avoid gradients, heavy shadows, smeared textures, and variable-pressure ink effects.
- Preserve a polished software aesthetic: playful, minimal, structured, and legible.

## Keywords To Preserve

- `Vector scribble style`
- `Wobbly vector lines`
- `Grid-aligned layout`
- `Structured`
- `Uniform stroke weight`
- `Isolated on a stark white background`

## Implementation Notes

- Treat this as a UI styling layer, not a replacement for layout discipline.
- When translating this into code, start with stable dimensions, aligned controls, and accessible typography, then add the hand-drawn character through borders, line textures, icons, and subtle irregularity.
- For generated assets, ask for clean vector-like UI forms with a consistent scribble filter. Do not ask for messy doodles, rough marker drawings, or whiteboard sketches.
